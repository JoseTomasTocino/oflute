Fundamentos teóricos

El sonido

El sonido es una vibración en forma de onda.

  • Frecuencia: oscilaciones por unidad de tiempo.
  • Amplitud: energía que transporta la onda.
  • Fase: desplazamiento respecto del origen.

Descomposición de sonidos

Los sonidos no suelen ser ondas puras, se componen de parciales.

Frecuencia fundamental: menor de los parciales. Dicta la altura general del sonido.

Los armónicos son parciales múltiplos de la frecuencia fundamental, enriquecen y caracterizan el sonido.

Objetivo: descomponer el sonido para obtener la frecuencia fundamental.

Herramientas de análisis armónico

Trabajan en el dominio de la frecuencia.

La transformada de Fourier es la más conocida: descompone una señal en sus componentes senoidales.

Algoritmo más habitual: FFT - Fast Fourier Transform.

Datos discretos: DFT - Discrete Fourier Transform.

Desarrollo

Objetivo:

Desarrollar un módulo que capture el sonido del micrófono, lo analice y detecte la nota que se está tocando.

Primer paso: capturar el audio

  • API de PulseAudio.
  • Abrimos un flujo de entrada.
  • Búffer de datos de sonido.
  • Procesamiento de los datos recogidos.

Segundo paso: analizar el sonido

  • Procesamos el búffer.
  • Aplicamos el algoritmo DFT.
  • Aislamos la frecuencia fundamental.
  • Comparación con tabla de frecuencias.
  • Se devuelve la nota detectada.

Objetivo:

Cargar y usar fuentes TrueType en Gosu

TrueType en Gosu

oFlute utiliza Gosu como sistema gráfico.

  • Open Source.
  • Ofrece renderizado 2D con aceleración por hardware.
  • Acceso a la entrada del usuario.
  • Multiplataforma: Windows, Linux, Mac.
  • API muy sencilla, orientada a objetos.
  • Desarrollo ágil.

TrueType en Gosu

Problema: Gosu no permite cargar fuentes TrueType en GNU/Linux.

Solución: implementar un módulo propio para fuentes TrueType.

El módulo se liberó. Ahora forma parte oficial de Gosu. 

// Used for custom TTF files
// Adapted from customFont class by Jose Tomas Tocino Garcia
class SDLTTFRenderer : boost::noncopyable

Objetivo:

Conseguir interfaces amigables y dinámicas

Interfaces con animaciones dinámicas

Problema: conseguir interfaces amigables, dinámicas, fluidas y minimalistas.

Solución: desarrollo de un sistema de animaciones de movimiento dinámicas.

Movimientos de aceleración, deceleración, uniformes, etc. sobre un número arbitrario de atributos.

Basadas en las ecuaciones de Robert Penner, liberadas bajo licencia BSD.

Objetivo:

Internacionalización de oFlute

Internacionalización de Gosu

Problema: internacionalizar el proyecto, con miras a otros países/culturas.

Solución: GNU Gettext como sistema de internacionalización.

Es el sistema de i18n más popular: multitud de herramientas, gran soporte y comunidades de traducción colaborativa, como Launchpad.

Publicación de Traducción de proyectos con GNU Gettext en 15 minutos.
